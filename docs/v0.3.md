# Structural Precedence Ledger v0.3

## Structure Lineage Graph

## Status

Active specification.

## Purpose

v0.3 introduces a graph representation for structural lineage.

v0.1 records the origin and precedence of one structure.

v0.2 assigns stable canonical identities to multiple structures.

v0.3 connects those structures through explicit,
auditable relationships.

## Core model

```text
Canonical Structure
        │
        ├── derived from
        ├── extends
        ├── specializes
        ├── implements
        ├── integrates
        ├── translates
        ├── converges with
        └── supersedes

Each structure is represented as a node.

Each relationship is represented as an edge.

Edge direction

Directed edges are read as:

source relationship_type target

Example:

Structure Lineage Graph
        extends
Canonical Structure Registry

In YAML:

source_node_id: "NOD-0003"
target_node_id: "NOD-0002"
relationship_type: "extends"

For lineage relationships, the source is generally the later
or derivative structure.

The target is generally the earlier or referenced structure.

Node kinds
canonical_structure

A structure registered in the Canonical Structure Registry.

It must include:

canonical_structure_id
canonical English name;
first-publication timestamp;
source URI.

The validator compares its identity and timestamp with the
registry.

external_structure

A structure published outside the canonical registry.

It must include:

external_structure_id
name;
first-publication timestamp;
source URI.

External structures allow later research, products, standards,
or implementations to be compared with registered structures.

Relationship types
Lineage relationships
derived_from
extends
specializes
generalizes
implements
integrates
translates
supersedes

These relationships are directed and subject to chronology
and cycle validation.

Influence relationships
direct_influence
probable_influence

A direct_influence edge requires documented direct evidence.

A probable_influence edge requires at least indirect or
documented access evidence.

Convergence relationships
independent_convergence
partial_convergence
conceptual_analogy
related_to

These relationships may be structurally symmetric.

Duplicate reverse edges are rejected.

Influence boundaries

Every edge contains an influence audit:

influence_audit:
  evidence_of_access: "none"
  evidence_of_citation: "none"
  evidence_of_direct_influence: "none"
  audit_note: >-
    Similarity exists, but direct influence is not established.

Similarity must not be silently upgraded into influence.

Required comparison

Every edge records:

convergences:
  - "Both structures preserve a stable origin identity."

differences:
  - "Only one structure defines royalty routing."

This prevents one shared word or superficial resemblance from
being treated as complete structural equivalence.

Semantic validation

The v0.3 validator checks:

duplicate node IDs;
duplicate edge IDs;
duplicate canonical structure nodes;
duplicate external structure nodes;
duplicate evidence IDs;
node and edge ordering;
registry identity consistency;
registry timestamp consistency;
missing source or target nodes;
self-edges;
duplicate directed edges;
duplicate symmetric relationships;
impossible chronological direction;
assertions made before publication;
invalid direct influence claims;
invalid probable influence claims;
independent convergence conflicts;
directed lineage cycles.
Cycle policy

Lineage relationships must not form a directed cycle.

Invalid example:

A extends B
B extends C
C extends A

This is rejected because the origin and derivative direction
becomes impossible to interpret.

Symmetric relations such as related_to are not included in
lineage-cycle detection.

External convergence example
node_id: "NOD-0100"
node_kind: "external_structure"
external_structure_id: "EXT-2026-0001"
name: "External provenance architecture"
organization: "Example Laboratory"
first_publication_at: "2026-09-01T00:00:00Z"
source_uri: "https://example.org/research"

An external node does not become part of the canonical registry
merely by appearing in a graph.

Non-goals

v0.3 does not:

prove legal ownership;
infer influence from similarity;
automatically discover every derivative;
calculate royalty shares;
resolve disputed historical claims;
replace citation systems;
replace patents or contracts;
create future prediction records.

Prediction and outcome auditing is reserved for v0.4.

Core principle

A structure is not only a point of origin.
It is a node in a traceable lineage.
